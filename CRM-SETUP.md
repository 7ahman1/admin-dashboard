# CRM Dashboard Setup Guide

## Overview
This CRM Dashboard provides lead tracking and management features for your AI Solutions business. It integrates seamlessly with your existing Supabase database.

## Features
✅ **Dashboard** - View key metrics and recent leads at a glance  
✅ **Lead Management** - Create, read, update, and delete leads  
✅ **Status Tracking** - Track leads through different stages (Pending, Contacted, Qualified, Won, Lost)  
✅ **Auto-Sync Bookings** - Website bookings automatically synced to CRM as leads  
✅ **Manual Sync** - Click "Sync Bookings" button to manually sync any time  
✅ **Search & Filter** - Find leads by status, name, email, or service  
✅ **Reports & Analytics** - View conversion rates, leads by service, and more  
✅ **Export to CSV** - Download your leads data  
✅ **Responsive Design** - Works on desktop and mobile devices  
✅ **Local Storage Fallback** - Works even if Supabase is unavailable  

## Installation & Setup

### Step 1: Update Bookings Table (Add new columns)

If your bookings table already exists, add these new columns by running this SQL in Supabase:

```sql
-- Add new columns to bookings table
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS phone TEXT;
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS booking_date DATE;
ALTER TABLE bookings ADD COLUMN IF NOT EXISTS booking_time TIME;
```

If this is a fresh installation, use this complete SQL:

```sql
CREATE TABLE bookings (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT,
  service TEXT NOT NULL,
  booking_date DATE,
  booking_time TIME,
  remarks TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_bookings_email ON bookings(email);
CREATE INDEX idx_bookings_service ON bookings(service);
```

### Step 2: Create the Leads Table in Supabase

1. Go to your Supabase Dashboard: https://app.supabase.com
2. Select your project
3. Go to **SQL Editor** on the left sidebar
4. Click **New Query**
5. Copy and paste this SQL:

```sql
CREATE TABLE leads (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT,
  service TEXT NOT NULL,
  status TEXT DEFAULT 'pending',
  company TEXT,
  budget DECIMAL,
  remarks TEXT,
  followup_date DATE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create index for faster queries
CREATE INDEX idx_leads_status ON leads(status);
CREATE INDEX idx_leads_service ON leads(service);
CREATE INDEX idx_leads_email ON leads(email);
```

6. Click **Run** to execute
7. Verify the table appears in the **Tables** section on the left

### Step 3: Set Up Row Level Security (Optional but Recommended)

For production use, set up RLS policies:

```sql
-- Enable RLS
ALTER TABLE leads ENABLE ROW LEVEL SECURITY;

-- Allow all operations (for public access)
CREATE POLICY "Enable read access for all users" ON leads FOR SELECT USING (true);
CREATE POLICY "Enable insert access for all users" ON leads FOR INSERT WITH CHECK (true);
CREATE POLICY "Enable update access for all users" ON leads FOR UPDATE USING (true);
CREATE POLICY "Enable delete access for all users" ON leads FOR DELETE USING (true);
```

## Auto-Sync Bookings to Leads

The CRM automatically syncs all bookings from your website's booking form to the leads table.

### How It Works:
1. **Auto-Sync**: Every 30 seconds, the CRM checks for new bookings from the website
2. **Convert**: Bookings are automatically converted to leads with "Pending" status
3. **Avoid Duplicates**: The system checks to prevent duplicate entries
4. **Manual Sync**: Click the "Sync Bookings" button anytime to manually sync

### Booking → Lead Fields:
| Booking Field | Lead Field |
|---------------|-----------|
| name | name |
| email | email |
| phone | phone |
| service | service |
| booking_date | remarks (with booking details) |
| booking_time | remarks (with booking details) |
| remarks | remarks (appended with "From booking:" prefix) |
| created_at | created_at |
| (auto-generated) | status = "pending" |

### Example Workflow:
1. Customer submits booking on your website
2. Booking saved to Supabase `bookings` table
3. CRM auto-syncs every 30 seconds
4. New lead appears in CRM dashboard automatically
5. You can then update status, add company info, set follow-up dates, etc.

### Fallback: Local Storage
- If Supabase is unavailable, data is stored locally in browser
- Data persists until browser cache is cleared
- Automatically syncs to Supabase when connection is restored

## Troubleshooting

### Issue: "No leads found" but I added leads
**Solution**: 
- Check your internet connection
- Verify Supabase table exists
- Check browser console for errors (F12)
- Clear browser cache and reload

### Issue: Bookings not syncing to CRM
**Solution**:
- Click "Sync Bookings" button to manually trigger sync
- Check that bookings table exists in Supabase
- Verify bookings are being saved to the database first
- Check browser console for sync errors (F12)
- Auto-sync runs every 30 seconds - wait a minute and refresh

### Issue: Duplicate leads appearing
**Solution**:
- This is normal if you manually add a lead with same email+service as a booking
- The system checks for exact duplicates but won't catch partial matches
- Consider having a workflow where either you manually add OR bookings auto-sync

### Issue: Can't see the Edit button
**Solution**:
- Make sure you're in the "All Leads" section
- The table might be too narrow on mobile - try landscape orientation

### Issue: Changes aren't saving
**Solution**:
- Check your internet connection
- Verify Supabase API key is correct in `supabase.js`
- Check browser console for error messages (F12)

### Issue: Export button not working
**Solution**:
- Make sure you have leads to export
- Use Chrome, Firefox, or Edge (not all browsers support file download)

## Security Notes

⚠️ **Important**: Your Supabase API key is public (anon key). For production:
1. Use Row Level Security (RLS) policies
2. Implement user authentication
3. Don't store sensitive data in lead fields
4. Regular backups of your Supabase database

## API Reference

The dashboard uses your existing Supabase connection from `supabase.js`:

```javascript
client = supabase.createClient(supabaseUrl, supabaseKey);
```

Main operations:
- `INSERT`: Add new leads
- `SELECT`: Fetch leads
- `UPDATE`: Modify lead details
- `DELETE`: Remove leads

All operations include error handling and fallback to localStorage.
